# Submissions

Official submissions are not open yet. This document defines the intended format so early contributors can prepare reproducible entries.

## Required Files

Each submission should provide:

- `submission.json`: metadata and commands
- `artifact/`: model artifact to be measured by `scripts/check_artifact_size.py`
- `src/`: training and inference code, or a link to an immutable public commit
- `README.md`: short method summary and reproduction notes
- `result-card.md`: method, budget, BPB, costs, failure modes, and reproducibility notes
- `costs.json`: machine-readable accounting input for `scripts/account_efficiency.py`

## Metadata Template

```json
{
  "name": "example-byte-unigram",
  "track": "local-student",
  "artifact_sha256": "...",
  "artifact_size_bytes": 1234,

  "optimizer_state_bytes": 123456,

  "selection_trials": 3,

    "python": "3.12"
  },
  "method_summary": "Byte unigram model with add-alpha smoothing."
}
```

## Reproducibility Requirements

- Commands must run from a clean clone.
- Dependencies must be pinned or described precisely.
- Training and evaluation data sources must be named and versioned.
- Artifacts must be generated by the supplied commands.
- Evaluation must not call external model APIs or fetch hidden state.
- Optimizer, selection, scoring, and update-budget overhead must be reported.
- Failure modes and negative or mixed results should be documented, not hidden.

## Review Process

Maintainers should check:

- artifact size
- accounting report
- deterministic or statistically stable scoring
- validation split compliance
- license compatibility
- clear attribution for borrowed techniques

Submissions that are promising but do not satisfy record-track rules may be listed in the Open Exploration track.

Use `templates/submission.json` and `templates/result-card.md` as the current research-track starting point.
